Purpose of application

The SVoC (Single View of Customer) technology solution is a central customer database dynamically built by matching data relating to a single customer from various source systems into customer composite view. It has an inbuilt algorithm to compare the new incoming data and update the DB with the latest customer data and keep a track of the changes done.

SVoC is an IBM MDM product that has been tailored to Boots requirements, hosted via WebSphere.

Initially Boots didnt haveany central customer database for AdCard & non-AdCard, so SVoC application was developed to store overall updated customer data for Boots landscape.

Overview of Functionality and Brief Descriptions of Boots SVoC Application

Major Functionalities of SVoC:

SVoC acts as a customer master data for upstream and downstream applications, which stores the most updated customer data, that is been captured by multiple source systems.

Key functionalities of SVoC are:

  • Receive Data from multiple source systems
  • Perform the Data validation and data cleaning by a level of validations and rules set in the middleware as well built-in in SVoC
  • Store the Data in the Virtual Database
  • Run algorithm to map the data to the existing customer records to map the incoming record as new customer or existing customer
  • Store the details and create the composite view
  • Trigger the composite view and incremental details to Physical DB and map it to Consent data.
  • Create composite view with the most updated customer records and consent data
  • Send the customer data to various downstream systems (ABACUS, Hadoop), and in future the consent and customer data will also get transferred from SVoC to ABACUS, Hadoop, SAP CRM and Hearing Care

Business Functionalities:

  • Acts as Central Database for Boots
  • Stores Cleansed and most updated Customer Data
  • Save source Applications Processing time
  • Share Data to other applications for Analysis purposes
  • Provides Composite view of customers Data

Key Features:

Single view of customer and central customer database dynamically built by matching data using algorithms relating to a single customer from various source systems
